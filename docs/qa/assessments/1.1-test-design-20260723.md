# Test Design: Story 1.1 — Project Scaffold & Health Tool (stdio)

Date: 2026-07-23
Designer: Quinn (Test Architect)

## Test Strategy Overview

- Total test scenarios: 6
- Unit tests: 3 (50%)
- Integration tests: 2 (33%)
- Doc/manual checks: 1 (17%)
- Priority distribution: P0: 4, P1: 2

| ID | Level | Priority | Test | Justification |
|----|-------|----------|------|---------------|

### AC2: Test harness runnable, ≥1 passing smoke test
| ID | Level | Priority | Test | Justification |
|----|-------|----------|------|---------------|

### AC3: Server runs over stdio, discoverable by MCP client
| ID | Level | Priority | Test | Justification |

### AC4: Health tool returns name/version/status with clear schema
| ID | Level | Priority | Test | Justification |
|----|-------|----------|------|---------------|
| 1.1-UNIT-002 | Unit | P1 | `server_info()` fields = name, version, status='ok' | Pure logic, cheapest level |

| ID | Level | Priority | Test | Justification |
|----|-------|----------|------|---------------|

| ID | Level | Priority | Test | Justification |

| ID | Level | Priority | Test | Justification |
|----|-------|----------|------|---------------|
| 1.1-UNIT-003 | Unit | P0 | Guard: no `print(` in `src/` | Mitigates OPS-001 (stdio corruption) |

## Risk Coverage

| Risk | Covered by |
|------|-----------|
| TECH-001 (SDK API mismatch) | 1.1-INT-001, 1.1-INT-002 |
