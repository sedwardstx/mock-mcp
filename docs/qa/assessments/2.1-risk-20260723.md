## Risk Matrix

| DATA-001 | Non-deterministic load order (dict/glob) breaks repeatability | Medium (2)  | Medium (2) | 4     | Medium   |

## Mitigations

- **DATA-001:** Sort scenarios by `scenario_id`; iterate files in sorted order; no `random`/clock. Determinism test loads twice, asserts identical order + content.
- **TECH-001:** Model mirrors architecture data-models exactly; telemetry container optional so Epic 3 adds data, not schema. Extensibility validated by loading varied samples.
- **DATA-002:** Pydantic strict validation; loader raises a clear error naming file+field; negative test.
- **DATA-003:** Loader enforces global unique ticket ids; duplicate-id negative test.
