# Story 2.4: Ticket ‚Üí Resource Pivot Tool

## Status

Done

## Story

**As a** student's AI agent,
**I want** to retrieve the Azure resource(s) and subscription associated with a ticket,
**so that** I can pivot from the reported incident to the resources whose telemetry I'll investigate.

## Acceptance Criteria

1. A tool (e.g., `get_ticket_resources`) accepts a `TICKET-XXXXXXXX` id and returns the associated Azure resource(s) ‚Äî including resource type (e.g., VM, VMSS), resource identifier, and subscription context.
2. Returned resource identifiers are consistent with those the Epic 3 telemetry tools will accept as scoping input.
3. A ticket with no/unknown resource, or a non-existent ticket id, returns a clear structured response rather than an error.
4. The tool exposes a clear MCP schema and description.
5. Integration tests assert the pivot returns the correct, deterministic resource(s) for known sample scenarios.

## Tasks / Subtasks

- [x] Task 1: Repository resource lookup (AC: 1, 3)
  - [x] `Repository.get_resources(ticket_id) -> list[AzureResource] | None` (None when the ticket does not exist)
- [x] Task 2: `get_ticket_resources` tool (AC: 1, 2, 3, 4)
  - [x] Validate id ‚Üí `invalid_request`; unknown ticket ‚Üí `not_found`; ticket with empty resources ‚Üí `ok` with empty list; else `ok` with resources
  - [x] Return full `AzureResource` (resource_id, resource_type, name, resource_group, subscription_id, location, instances)
- [x] Task 3: Register tool in server (AC: 1)
- [x] Task 4: Tests (AC: 5)
  - [x] Unit: repository get_resources (known, unknown); Integration: pivot ok / not_found / invalid via MCP client

## Dev Notes

### Design [Source: architecture/components.md#Repository] [Source: architecture/data-models.md#AzureResource]
- `AzureResource` fields: `resource_id` (ARM id), `resource_type`, `name`, `resource_group`, `subscription_id`, `location`, `instances` (VMSS instance ids; empty for VM).
- **AC2 (critical):** the returned `resource_id` is the exact ARM id the Epic 3 telemetry tools will accept as their scoping input ‚Äî do not transform it. [Source: architecture/database-schema.md]
- New tool module `tools/resources.py`; return a structured `ResourceResult` (`status`, `resources`, `message`). [Source: architecture/error-handling-strategy.md#Business-Logic-Errors]

### Error Handling
- Invalid id format ‚Üí `invalid_request`; unknown ticket ‚Üí `not_found`; empty resource list ‚Üí `ok` with `resources: []`. Never raise. [Source: architecture/coding-standards.md#Critical-Rules]
