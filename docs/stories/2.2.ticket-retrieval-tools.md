# Story 2.2: Ticket Retrieval Tools (List & Get)

## Status

Review

## Story

**As a** student's AI agent,
**I want** to list tickets and fetch a single ticket by its ID,
**so that** I can discover incidents to work and read the full details of one.

## Acceptance Criteria

1. A `get_ticket` tool accepts a `TICKET-XXXXXXXX` id and returns the full ticket detail (symptom, product, severity, status, persona, affected resource reference(s), timestamps).
2. A `list_tickets` tool returns a summary list of tickets and supports pagination or a bounded result size suitable for an agent context.
3. Requesting a non-existent or malformed ticket id returns a clear, structured "not found"/validation response rather than an error/stack trace.
4. Both tools expose clear MCP schemas and descriptions sufficient for an agent to select and call them without external docs.
5. Integration tests invoke both tools via an MCP client and assert correct, deterministic results against known sample scenarios.

## Dev Notes

### Repository & Tool Design [Source: architecture/components.md#Repository] [Source: architecture/components.md#MCP-Surface]
- **Repository** wraps the immutable `Dataset` from Story 2.1. Methods for this story: `get_ticket(ticket_id) -> Ticket | None`; `list_tickets(offset, limit) -> (list[Ticket], total)`. (Search/resource/telemetry methods added in later stories.)
- **Tools** are thin handlers over the repository; return structured Pydantic results. [Source: architecture/high-level-architecture.md]
- **Startup:** load the dataset once in `build_server()` (fail-fast) and inject the repository into tool registration. [Source: architecture/components.md#Fixture-Loader-Validator]

### Error Handling [Source: architecture/error-handling-strategy.md#Business-Logic-Errors]
- Not found → structured `{status: "not_found", message, ...}`.
- Malformed id → structured `{status: "invalid_request", message}` (reuse `TICKET_ID_RE` from models).
- Tools never raise past the handler. [Source: architecture/coding-standards.md#Critical-Rules]
