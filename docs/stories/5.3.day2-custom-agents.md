## Story

**As an** instructor,
**I want** the specialist and coordinator custom agents wired to this server's tools,
**so that** participants can run the Day-2 agent + coordinator labs.

## Acceptance Criteria

1. Three specialist agents exist — compute-diagnostics, network-diagnostics, controlplane-diagnostics — each referencing this server and naming the specific telemetry tools it uses.
2. A `support-triage` coordinator agent (tools: ['agent']) routes to the three specialists and offers a "Draft the resolution" handoff.
3. A minimal `ticket-writer` agent exists so the coordinator handoff resolves out of the box.
4. Agent frontmatter follows the workbook's conventions; specialist names exactly match the coordinator's `agents:` list.

## Dev Notes

### Agent conventions [Source: docs/external/Participant_Workbook.md] (Appendix A; Lab 5/6)
- Frontmatter fields: `description`, `name`, `tools`, optional `model`, `agents`, `handoffs`, `user-invocable`.
- **`tools:` references the MCP server by its `mcp.json` key** `contoso-support` (per architecture D1) — the persona body names the specific tools. [Source: architecture/workshop-integration-epic-5.md]
- Specialists also use `codebase`. Coordinator uses `tools: ['agent']` + `agents: [...]`.

### Domain map [Source: prd/epic-5-workshop-integration.md]
- compute-diagnostics → `query_compute_host_logs`, `query_compute_guest_logs` (VM/VMSS host+guest; cite TICKET-10000026 multi-round).
- network-diagnostics → `query_network_logs` (NSG/connectivity; port 1433 example).
- controlplane-diagnostics → `query_arm_traces` (+ `search_known_issues`).
- support-triage → classify Compute/Network/Control-plane, delegate, synthesize; handoff to ticket-writer.

### Server tools [Source: architecture/components.md#MCP-Surface]
- The 8 diagnostic tools + `search_known_issues`; all surfaced under the `contoso-support` server key.

### File Locations [Source: architecture/workshop-integration-epic-5.md]
- `.github/agents/{compute,network,controlplane}-diagnostics.agent.md`, `support-triage.agent.md`, `ticket-writer.agent.md`.
